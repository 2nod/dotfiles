# 返信フォーマット

レビューコメントへ返信するときに読む。

## 基本方針

原則として 1 review comment に対して 1 reply を対応させる。
commit は追跡しやすい単位にする。
複数コメントが同じ原因を指している場合や、既存 commit を amend するほうが自然な場合は、1 commit にまとめてよい。
返信では次を明確にする。

- 何を変更したか
- なぜその変更で指摘に対応できるか
- 対応 commit
- あえて変更しなかった点がある場合はその理由
- 何を検証したか

コメントに補足説明、詳細欄、自動レビューの追加観点などが含まれる場合は、それらも同じ指摘の一部として読み、返信前に対応漏れがないか確認する。

## 標準フォーマット

```md
対応しました。

指摘の通り、対象処理を利用者側の呼び出し方に近い経路で確認するように変更しました。これにより、単体の実装だけでなく周辺の契約も含めて検証できます。

対応 commit: [`abc1234`](https://github.com/OWNER/REPO/commit/abc1234)

確認: `yarn workspace ... test ...`
```
