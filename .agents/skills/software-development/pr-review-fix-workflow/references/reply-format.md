# 返信方針

レビューコメントへ返信するときに読む。

## 基本方針

原則として 1 review comment に対して 1 reply、1 reply に対して 1 commit を対応させる。返信では次を明確にする。

- 何を変更したか
- なぜその変更で指摘に対応できるか
- 対応 commit
- あえて変更しなかった点がある場合はその理由
- 何を検証したか

コメントに補足説明、詳細欄、自動レビューの追加観点などが含まれる場合は、それらも同じ指摘の一部として読み、返信前に対応漏れがないか確認する。

標準の返信文は `templates/review-reply.md` を構造として使う。
